## <a name="a-disks-status-is-not-initialized-or-the-disk-is-missing"></a>디스크가 누락 되었는지 또는 디스크의 상태가 초기화 되지 않았습니다.

![디스크 관리를 초기화 해야 하는 알 수 없는 디스크를 표시 합니다.](media/uninitialized-disk.PNG)

**원인:** 파일 탐색기에 나타나지 않으며 디스크 관리에 나열 되어 있는 디스크의 경우 *초기화 되지 않았습니다*, 디스크에서 올바른 디스크 서명 없는 때문일 수 있습니다. 기본적으로이 디스크 되지 초기화 및 포맷, 또는 서식을 드라이브 손상 된 어떤 이유로 든 의미 합니다. 

하드웨어 문제 또는 문제, 연결 된 디스크 것 있지만 잠시 단락에서는 드리겠습니다는 이기도 합니다.

**해결 방법:**   드라이브는 완전히 새로운 하기만 초기화를 위해 모든 데이터를 지운 경우 솔루션을 쉽게-참조 [새 디스크 초기화](initialize-new-disks.md)합니다. 그러나 이미 시도한이 가능성이 있고 작동 하지 않았습니다. 중요 한 파일의 전체 디스크를 보았다면 및 초기화 하 여 디스크 지우기 않으려는 합니다.

디스크 누락 될 수도 있습니다는 일반적인 이유는 디스크 실패 하기 때문에 초기화에 실패 하는 이유 중 여러 가지가 있습니다. 수많은 작업을 실패 한 디스크를 해결 하려면 할 수 있지만 것을 확인할 경우 수 다시 작동 하려면 일부 단계는 다음과 같습니다. 디스크 후 다음이 단계 중 하나를 작동 하는 경우 하지 다음 단계를 사용만 다시 시작, 복귀를 기념 하기를 및 백업을 업데이트할 수 있습니다.

1. 디스크 관리에서 디스크를 살펴봅니다. 표시 되는 경우 *오프 라인* 다음과 같이를 마우스 오른쪽 단추로 클릭 하 고 선택 **Online**합니다.

    ![오프 라인으로 표시 되는 디스크](media/offline-disk.png)
2. 디스크와 디스크 관리에 표시 되 면 *Online*,으로 표시 되는 주 파티션이 있고 *정상*표시 된 것 처럼 여기에서 좋은 징후입니다.

    ![디스크 정상 볼륨을 사용 하 여 온라인으로 표시](media/healthy-volume.png)
    - 파티션을 파일 시스템에는 있지만 (예를 들어 e:) 드라이브 문자 없음, 참조 [드라이브 문자 변경](change-a-drive-letter.md) 드라이브 문자를 수동으로 추가 합니다.
    - 파티션을 마우스 오른쪽 단추로 클릭 하 고 선택 파일 시스템 (NTFS, ReFS, FAT32 또는 exFAT) 없는 경우 디스크가 비어 알고 **형식**합니다. 디스크를 포맷 되므로 실행 하지 마십시오. 대신-디스크에서 파일 복구, 다음 단계를 건너 뛰 세요 하고자 하는 경우 모든 데이터가 지워집니다.
3. 디스크에 디스크를 분리 하는 외부에 있는 경우 연결에서 다시 선택한 후 **동작** > **디스크 다시 검사**합니다. 
4. PC를 종료 하, 외부 하드 디스크 (전원 코드를 사용 하 여 외부 디스크 경우)를 해제 한 PC와 디스크 다시 켭니다.
    Windows 10에서 PC를 해제 하려면 시작 단추를 선택, 전원 단추를 선택한 후 **종료**합니다.
5. (허브)에 없는 PC에서 직접 수행 되며 다른 USB 포트에 디스크를 연결 합니다.
    USB 디스크 경우에 따라 일부 포트에서 충분 한 전원을 가져오기 또는 특정 포트를 사용 하 여 다른 문제가 하지 마세요. 따라서 있다면 몇 가지 서로 다른 포트를 시도 하지만 경우에 따라 차이가 PC에서 포트 USB 허브를 사용 하 여 특히 합니다.
6. 다른 케이블을 테스트 합니다.
    미친 듯이 바쁜 게를 보일 수 있지만 케이블 훨씬 실패 하므로에 디스크를 연결할 다른 케이블을 사용 하 여를 시도 하세요. 내부 디스크 데스크톱 PC에 있는 경우 아마도 해야 PC 케이블을 전환 하기 전에 종료-세부 정보에 대 한 사용자 PC의 설명서를 참조 하십시오.
7. 문제에 대 한 장치 관리자를 확인 합니다.
    키를 눌러 및 보관 (또는 마우스 오른쪽 단추로 클릭) 시작 단추 상황에 맞는 메뉴에서 장치 관리자를 선택 합니다. 또는 기타 문제 옆에 있는 느낌표가 있는 장치를 찾습니다 하, 장치를 두 번 클릭 한 다음 해당 상태를 읽습니다.

    목록은 다음과 같습니다 [장치 관리자에서 오류 코드](https://support.microsoft.com/help/310123/error-codes-in-device-manager-in-windows), 이지만 선택 하는 경우에 따라 작동 하는 문제가 있는 장치를 마우스 오른쪽 단추로 클릭 한 가지 방법은 **제거 장치**를 차례로 **동작**  >  **하드웨어 변경 사항 검색**합니다.

    ![알 수 없는 USB 장치를 표시 하는 장치 관리자](media/device-manager.PNG)
8. 다른 PC에 디스크를 연결 합니다.
    
    디스크를 다른 PC에서 사용할 수 없을 경우 디스크와 PC 하지 진행 잘못 된 것은 좋은 징후 것입니다. 재미를 알게 됩니다. 몇 가지 자세한 단계에 시도할 수 있습니다 [외부 USB 드라이브 "전에 초기화 해야 디스크 논리 디스크 관리자가 액세스할 수 있습니다" 오류가](https://social.technet.microsoft.com/Forums/windows/en-US/2b069948-82e9-49ef-bbb7-e44ec7bfebdb/forum-faq-external-usb-drive-error-you-must-initialize-the-disk-before-logical-disk-manager-can?forum=w7itprohardware), 있지만 검색에 도움을 요청 하는 시간을 [Microsoft 커뮤니티](https://answers.microsoft.com/en-us/windows) 사이트를 사용 하거나 디스크 제조업체에 문의 합니다.

    실패 한 디스크에서 데이터를 복구 해 볼 수 있는 앱 밖에도 확인할 수 없는 경우 작동 합니다 또는 파일에 매우 중요 한 경우 요금 지불할 수 있습니다 이러한 복구를 시도 하는 데이터 복구 환경을 합니다. 작동 하는 것을 찾을 경우 아래 의견 섹션에 알려 주십시오.

> [!IMPORTANT]
> 디스크는 정기적으로 중요 한 모든 파일을 백업 해야 하므로 매우 자주 실패 합니다. 때때로 나타나지 또는 오류를 제공 하는 디스크에 있는 경우 백업 메서드를 다시 확인 하 라는 알림이 참고 하십시오. 작은 뒤 라면-겠죠 있는 경우 확인 됩니다. 최고의 백업 솔루션 이므로 하나를 사용 하기를 찾을 지 키십시오를 것이 좋습니다.

## <a name="a-basic-or-dynamic-disks-status-is-unreadable"></a>기본 또는 동적 디스크의 상태를 읽을 수 없음

**원인:**  기본 또는 동적 디스크에 액세스할 수 없는 및 하드웨어 오류, 손상 또는 I/O 오류가 발생 했을 수 있습니다. 시스템의 디스크 구성 데이터베이스의 디스크 복사본이 손상되었을 수 있습니다. **읽을 수 없음** 상태를 표시하는 오류 아이콘이 디스크에 표시됩니다.

스핀업을 하는 동안 또는 디스크 관리에서 시스템의 모든 디스크를 다시 검사할 때 디스크가 **읽을 수 없음**으로 표시될 수도 있습니다. 경우에 따라 읽을 수 없는 디스크가 실패했고 복구할 수 없게 됩니다. 동적 디스크의 경우 **읽을 수 없음** 상태는 주로 전체 디스크 오류보다는 디스크 일부분의 손상 또는 I/O 오류로 인해 발생합니다.

**해결 방법:**  디스크 다시 검사 또는 디스크 상태가 변경 되는지 확인 하려면 컴퓨터를 다시 시작 합니다. 에 설명 된 문제 해결 단계를 시도할 수도 [디스크의 상태가 초기화 되지 않았습니다 또는 디스크가 누락 되었는지 완전히](#a-disks-status-is-not-initialized-or-the-disk-is-missing)합니다.

## <a name="a-dynamic-disks-status-is-foreign"></a>외부 동적 디스크의 상태가입니다.

**원인:**  는 **외래** 상태 PC 다른 컴퓨터에서 로컬 컴퓨터에 동적 디스크를 이동할 때 발생 합니다. **외부** 상태를 표시하는 경고 아이콘이 디스크에 표시됩니다.

경우에 따라 이전에 시스템에 연결되었던 디스크가 **외부** 상태로 표시될 수 있습니다. 동적 디스크 구성 데이터는 모든 동적 디스크에 저장됩니다. 따라서 모든 동적 디스크에 오류가 발생하면 시스템이 소유하는 디스크에 관한 정보가 상실됩니다.

**해결 방법:**  디스크에 데이터를 액세스할 수 있도록 컴퓨터의 시스템 구성에 디스크를 추가 합니다. 디스크를 컴퓨터의 시스템 구성에 추가하려면 외부 디스크를 가져옵니다(디스크를 마우스 오른쪽 단추로 클릭한 다음 **외부 디스크 가져오기** 클릭). 디스크를 가져올 때 외부 디스크의 모든 기존 볼륨이 표시되고 액세스가 가능합니다. 

## <a name="a-dynamic-disks-status-is-online-errors"></a>동적 디스크의 상태가 온라인 (오류)

**원인:**  동적 디스크는 디스크의 영역에 대 한 I/O 오류입니다. 오류가 있는 동적 디스크에 경고 아이콘이 표시됩니다.

**해결 방법:**   I/O 오류 임시 경우에 반환 하기 위해 디스크를 다시 활성화 **Online** 상태입니다.

## <a name="a-dynamic-disks-status-is-offline-or-missing"></a>동적 디스크의 상태가 오프 라인 또는 없음

**원인:**  는 **오프 라인** 동적 디스크가 손상 되었거나 일시적으로 사용할 수 있습니다. 오프라인 동적 디스크에 오류 아이콘이 표시됩니다.

디스크 상태가 **오프라인**이고 디스크의 이름이 **Missing**으로 변경된 경우 디스크가 최근에는 이용할 수 있었지만 더 이상 위치하거나 식별될 수 없습니다. 없음 상태의 디스크는 손상되었거나 전원이 꺼졌거나 연결이 끊어졌을 수 있습니다.

**해결 방법:** 에 오프 라인 및 온라인 다시 누락 된 디스크를 표시 합니다.

1. 디스크, 컨트롤러 또는 케이블 문제를 해결합니다. 
2. 실제 디스크가 켜져 있고 컴퓨터에 연결되어 있는지 확인합니다. 
3. 그 다음 **디스크 다시 활성화** 명령을 사용하여 디스크를 온라인 상태로 설정합니다.
4. 에 설명 된 문제 해결 단계를 시도 [디스크의 상태가 초기화 되지 않았습니다 또는 디스크가 누락 되었는지 완전히](#a-disks-status-is-not-initialized-or-the-disk-is-missing)합니다.
5. 디스크 상태가 여전히 **오프라인**이고 디스크 이름이 여전히 **Missing**인 경우 디스크에 복구할 수 없는 문제가 있음을 확인합니다. 디스크를 마우스 오른쪽 단추로 클릭한 다음 **디스크 제거**를 클릭하여 시스템에서 디스크를 제거할 수 있습니다. 하지만 디스크를 제거하지 전에 디스크의 모든 볼륨(또는 미러)를 삭제해야 합니다. 전체 볼륨 대신 미러를 제거하여 디스크의 미러링된 볼륨을 저장할 수 있습니다. 볼륨을 삭제하면 볼륨의 데이터가 손상됩니다. 따라서 디스크가 영구적으로 손상되어 사용할 수 없음을 확신하는 경우에만 디스크를 제거해야 합니다.

**디스크를 오프 라인 이며 디스크 이름은 여전히 상태로 만드는 \# (없습니다) 다시 온라인으로 다음 절차 중 하나 이상을 수행해 봅니다.**

1. 디스크 관리에서 디스크를 마우스 오른쪽 단추로 클릭하고 **디스크 다시 활성화**를 클릭하여 디스크를 다시 온라인 상태로 설정합니다. 디스크 상태가 여전히 **오프라인**인 경우 케이블 및 디스크 컨트롤러를 확인한 다음 실제 디스크가 정상인지 확인합니다. 모든 문제를 수정하고 디스크를 다시 활성화합니다. 디스크 활성화에 성공한 경우 디스크의 모든 볼륨이 자동으로 **정상** 상태가 됩니다.
2. 이벤트 뷰어에서 "적절한 구성 복사본이 없음"과 같은 디스크 관련 오류에 대한 이벤트 로그를 확인합니다. 이벤트 로그에 이 오류가 포함된 경우 [Microsoft 기술 지원 서비스](https://msdn.microsoft.com/library/aa263468(v=vs.60).aspx)에 문의합니다.

3. 디스크를 다른 컴퓨터로 이동해 봅니다. 디스크를 다른 컴퓨터에서 **온라인**으로 설정할 수 있는 경우 문제는 디스크가 **온라인**으로 설정되지 않는 컴퓨터의 구성이 원인일 가능성이 높습니다.

4. 디스크를 동적 디스크가 있는 다른 컴퓨터로 이동해 봅니다. 컴퓨터에 있는 디스크를 가져온 다음 디스크를 **온라인**으로 설정되지 않는 컴퓨터로 이동합니다. 

## <a name="a-basic-or-dynamic-volumes-status-is-failed"></a>기본 또는 동적 디스크의 상태가 실패

**원인:**   기본 또는 동적 볼륨을 자동으로 시작할 수 없습니다, 디스크 손상 되었거나 파일 시스템이 손상 되었습니다. 디스크 또는 파일 시스템을 복구할 수 있는 경우를 제외하면 **실패** 상태는 데이터 손실을 나타냅니다.

**해결 방법:**

양이 기본 볼륨이 **실패** 상태:

- 기본 실제 디스크가 켜져 있고 컴퓨터에 연결되어 있는지 확인합니다.
- 에 설명 된 문제 해결 단계를 시도 [디스크의 상태가 초기화 되지 않았습니다 또는 디스크가 누락 되었는지 완전히](#a-disks-status-is-not-initialized-or-the-disk-is-missing)합니다.

볼륨이 **실패** 상태인 동적 볼륨인 경우:

-   기본 디스크가 온라인 상태인지 확인합니다. 그렇지 않은 경우 디스크를 **온라인** 상태로 다시 설정합니다. 성공한 경우 볼륨이 자동적으로 다시 시작되고 **정상** 상태로 돌아갑니다. 동적 디스크가 **온라인** 상태로 돌아가지만 동적 볼륨이 **정상** 상태로 돌아가지 않는 경우 수동으로 볼륨을 다시 활성화할 수 있습니다.
-   동적 볼륨이 오래된 데이터가 있는 미러 또는 RAID-5 볼륨인 경우 기본 디스크를 온라인으로 다시 설정해도 볼륨이 자동으로 다시 시작되지 않습니다. 현재 데이터를 포함한 디스크 연결이 끊어진 경우 우선 디스크를 온라인으로 설정합니다(데이터의 동기화된 상태 허용). 그렇지 않은 경우 수동으로 미러 또는 RAID-5 볼륨을 다시 시작한 다음 오류 검사 도구 또는 Chkdsk.exe를 실행합니다.
- 에 설명 된 문제 해결 단계를 시도 [디스크의 상태가 초기화 되지 않았습니다 또는 디스크가 누락 되었는지 완전히](#a-disks-status-is-not-initialized-or-the-disk-is-missing)합니다.

## <a name="a-basic-or-dynamic-volumes-status-is-unknown"></a>기본 또는 동적 디스크의 상태는 알 수 없음

**원인:**   는 **알 수 없는** 경우 상태가 (때문일, 바이러스) 볼륨에 대 한 부팅 섹터가 손상 되 고 볼륨에 데이터를 더 이상 액세스할 수 없습니다. **알 수 없음** 상태는 또한 새 디스크를 설치하지만 디스크 서명 만들기 마법사를 성공적으로 완료하지 않을 때 발생합니다.

**솔루션**  디스크를 초기화 합니다. 지침은 [새 디스크 초기화](initialize-new-disks.md)를 참조하세요.

## <a name="a-dynamic-volumes-status-is-data-incomplete"></a>동적 볼륨의 상태는 데이터가 완전 하지 않음

**원인:** 일부 이동 하면 전부가 아니라 다중 디스크 볼륨의 디스크. 이 볼륨을 포함하는 남은 디스크를 이동하고 가져오지 않는 한 이 볼륨의 데이터는 손실됩니다.

**해결 방법:**

1. 다중 디스크 볼륨을 구성하는 모든 디스크를 컴퓨터로 이동합니다.
2. 디스크를 가져옵니다. 디스크를 이동하고 가져오는 방법을 설명하는 지침은 [디스크를 다른 컴퓨터로 이동](move-disks-to-another-computer.md)을 참조하세요.

다중 디스크 볼륨이 더 이상 필요하지 않은 경우 디스크를 가져오고 디스크에 새 볼륨을 만들 수 있습니다. 이를 수행하려면:

1. **실패** 또는 **중복 실패** 상태인 볼륨을 마우스 오른쪽 단추로 클릭한 다음 **볼륨 삭제**를 클릭합니다.
2. 디스크를 마우스 오른쪽 단추로 클릭한 다음 **새 볼륨**을 클릭합니다.

## <a name="a-dynamic-volumes-status-is-healthy-at-risk"></a>동적 볼륨의 상태가 정상 (위험)

**원인:**   나타내지만 동적 볼륨은 현재 액세스할 수 있는 기본 동적 디스크에서 I/O 오류가 검색 되었습니다. 동적 디스크의 어떤 부분에서 I/O 오류가 감지된 경우 디스크의 모든 볼륨이 **정상(위험)** 상태로 표시되고 볼륨에 경고 아이콘이 표시됩니다.

볼륨 상태가 **정상(위험)** 일 때 기본 디스크의 상태는 주로 **온라인(오류)** 입니다.

**해결 방법:**  
1. 기본 디스크를 **온라인** 상태로 다시 설정합니다. 디스크가 **온라인** 상태로 다시 설정되면 볼륨은 **정상** 상태로 다시 설정됩니다. **정상(위험)** 상태가 유지되는 경우 디스크 오류가 있을 수 있습니다. 

2. 가능하면 데이터를 백업하고 디스크를 교체합니다. 

## <a name="cannot-manage-striped-volumes-using-disk-management-or-diskpart"></a>디스크 관리 또는 DiskPart를 사용 하 여 스트라이프 볼륨을 관리할 수 없습니다.

**원인:**   일부 타사 디스크 관리 제품은 LDM 사용 하지 않도록 설정할 수 있는 고급 디스크 관리에 대 한 Microsoft 논리 디스크 관리자 (LDM)를 대체 합니다.

**해결 방법:**   LDM 비활성화 타사 디스크 관리 소프트웨어를 사용 하는 경우 디스크 문제 해결에 대 한 지원을 대 한 타사 디스크 관리 소프트웨어 공급 업체에 문의 해야 합니다 있습니다 구성입니다.

## <a name="disk-management-cannot-start-the-virtual-disk-service"></a>디스크 관리에서 가상 디스크 서비스를 시작할 수 없습니다.

**원인:**   경우 원격 컴퓨터에서 가상 디스크 서비스 (VDS)를 지원 하지 않거나 Windows 방화벽에 의해 차단 되어 원격 컴퓨터에 대 한 연결을 설정할 수 없는 경우이 오류가 나타날 수 있습니다.

**해결 방법:**

1. 원격 컴퓨터가 VDS를 지원하는 경우 Windows Defender 방화벽이 VDS 연결을 허용하도록 구성할 수 있습니다. 원격 컴퓨터가 VDS를 지원하지 않는 경우 원격 데스크톱 연결을 사용하여 연결한 다음 원격 컴퓨터에서 디스크 관리를 직접 실행할 수 있습니다.
2. VDS를 지원하는 원격 컴퓨터에서 디스크를 관리하려면 로컬 컴퓨터(디스크 관리를 실행하는 컴퓨터) 및 원격 컴퓨터 모두에서 Windows Defender 방화벽을 구성해야 합니다.
3. 로컬 컴퓨터에서 Windows Defender 방화벽이 원격 볼륨 관리 예외를 설정하도록 구성합니다.

> [!NOTE]
> 원격 볼륨 관리 예외에는 Vds.exe, Vdsldr.exe 및 TCP 포트 135에 대한 예외가 포함됩니다.

> [!NOTE]
> 작업 그룹의 원격 연결은 지원되지 않습니다. 로컬 컴퓨터와 원격 컴퓨터 모두 도메인의 구성원이어야 합니다.
