## <a name="guidance-on-disabling-system-services-on-windows-server-2016-with-desktop-experience"></a>데스크톱 환경 포함 Windows Server 2016에서 시스템 서비스를 사용 하지 않도록 설정에 대 한 지침

적용 대상: Windows Server 2016

Windows 운영 체제는 중요 한 기능을 제공 하는 많은 시스템 서비스를 포함 합니다. 다른 서비스에 서로 다른 기본 시작 정책: 일부 기본적으로 시작 됩니다 (자동) 일부 필요한 경우 (수동) 및 일부 기본적으로 비활성화 되어 실행 하기 전에 명시적으로 설정 해야 합니다. 또한 일반 고객에 대 한 성능, 기능 및 보안 균형을 유지 하려면 각 서비스에 대 한 이러한 기본값은 신중 하 게 선택 되었습니다.

그러나 일부 기업 고객은 자신의 Windows Pc에 대 한 보안 중심 균형을 수도 및 공격을 줄일 수 있는 서버는 절대 최소값 화면 및 완벽 하 게 특정에 필요 하지 않은 모든 서비스를 사용 하지 않도록 설정할 수 있습니다 따라서 환경입니다. 이러한 고객에 게 Microsoft®는에 대 한 서비스 안전 하 게 해제할 수 있습니다이 목적을 위해 함께 제공 되는 지침을 제공 합니다.

지침 않는 데스크톱 환경 포함 Windows Server 2016에 대해서만 (최종 사용자에 게 데스크톱 대체본으로 사용)입니다. Windows Server 2019 부터는 이러한 지침 기본적으로 구성 됩니다. 시스템에서 각 서비스는 다음과 같이 분류 됩니다.

-   **비활성화 해야 합니다.** 보안 중심 엔터프라이즈 선호 하는이 서비스를 사용 하지 않도록 설정 및 기능 하지 않게 포함 가능성이 됩니다 (아래 추가 정보 참조).
- **사용 하지 않도록 설정 하려면 확인:** 이 사용 하지 않는 보안 중심 기업 안전 하 게 비활성화할 수 및이 서비스 전부는 아니지만 일부 기업에 게 유용한 기능을 제공 합니다.
- **해제 하지 않습니다.** 이 서비스를 사용 하지 않도록 설정 필수 기능에 영향을 줄 또는 특정 역할이 나 기능 정상적으로 작동 하지 못하도록 합니다. 따라서이 사용 해야 합니다.
-  **(지침 없음):** 이러한 서비스를 사용 하지 않도록 설정의 영향 완벽 하 게 평가 하지 않았습니다. 따라서 이러한 서비스의 기본 구성을 변경할 수 없습니다.


고객은 자신의 Windows Pc와 서버는 그룹 정책에서 보안 템플릿을 사용 하거나 PowerShell 자동화를 사용 하 여 선택한 서비스를 사용 하지 않도록 설정 구성할 수 있습니다. 경우에 따라 지침에 서비스 자체를 사용 하지 않도록 설정 하는 대신 직접 서비스의 기능을 해제 하는 특정 그룹 정책 설정이 포함 됩니다.

고객은 다음과 같은 서비스 및 데스크톱 환경 포함 Windows Server 2016에는 각 예약 된 작업 하지 않도록 하는 것이 좋습니다.

서비스: 
1. Xbox Live 인증 관리자
2. Xbox Live 게임 저장

예약 된 작업: 
1. \Microsoft\XblGameSave\XblGameSaveTask
2. \Microsoft\XblGameSave\XblGameSaveTaskLogon

(연결 된 Microsoft Excel 스프레드시트를 확인 하 여이 문서에서 설명 하는 모든 서비스에 대 한 정보에 액세스할 수도 있습니다. [데스크톱 환경 포함 Windows Server 2016의 시스템 서비스를 사용 하지 않도록 설정 하는 지침은](https://msdnshared.blob.core.windows.net/media/2017/05/Service-management-WS2016.xlsx))

### <a name="disabling-services-not-installed-by-default"></a>기본적으로 설치 되지 서비스를 사용 하지 않도록 설정

기본적으로 설치 되지 않은 서비스를 사용 하지 않도록 설정 하려면 정책 적용에 대해 권장 됩니다.
-  서비스의 기능이 설치 된 경우 일반적으로 필요 합니다. 서비스 또는 기능 설치 관리자 권한이 필요 합니다. 서비스의 시작 되지 기능 설치를 허용 하지 않습니다.
-  Microsoft Windows 서비스를 사용 하 여 차단 하는 비슷한 타사 이와 동등한 보안 위험은 높음 아마도 하나 설치 관리자 (또는 관리자가 아닌 경우에 따라) 중지 되지 않습니다.
-  초기 또는 기본이 아닌 Windows 서비스 (w3svc) 예를 들어 사용 하지 않도록 설정 하는 벤치 마크는 느낌을 주어 서 일부 감사자 잘못 된 기술 (예: IIS)는 본질적으로 안전 하지 않습니다 하 고 사용 하지 말아야 합니다.
-  기능 (및 서비스) 설치 되지 않은 경우이 추가 불필요 한 대량 확인 작업 하는 기준으로 합니다.

<br />
이 문서에 나열 된 모든 시스템 서비스에 대 한 수행 하는 두 테이블 열 및 설명은 Microsoft 권장 사항을 설정 및 해제 데스크톱 환경 포함 Windows Server 2016의 시스템 서비스를 제공 합니다.

| | |
|---|---|
|   **서비스 설명** |   Windows 원격 관리(WinRM) 서비스는 원격 관리용 WS-관리 프로토콜을 구현합니다. WS-관리는 원격 소프트웨어 및 하드웨어 관리에 사용되는 표준 웹 서비스 프로토콜입니다. WinRM 서비스는 네트워크에서 WS-관리 요청을 수신하고 이를 처리합니다. WinRM 서비스는 네트워크에서 수신하려면 winrm.cmd 명령줄 도구를 사용하는 수신기나 그룹 정책을 통해 구성해야 합니다. WinRM 서비스는 WMI 데이터에 대한 액세스를 제공하고 이벤트 수집을 허용합니다. 이벤트를 수집하고 구독하려면 이 서비스가 실행 중이어야 합니다. WinRM 메시지는 전송 방법으로 HTTP 및 HTTPS를 사용합니다. WinRM 서비스는 IIS에 종속되지 않지만 같은 시스템에서 IIS와 포트를 공유하도록 미리 구성되어 있습니다.  WinRM 서비스는 /wsman URL 접두사를 예약합니다. IIS와의 충돌을 방지하기 위해 관리자는 IIS에서 호스트하는 모든 웹 사이트가 /wsman URL 접두사를 사용하지 않는 것을 확인해야 합니다.
