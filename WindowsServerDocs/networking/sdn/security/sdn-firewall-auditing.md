# <a name="sdn-firewall-auditing"></a>SDN 방화벽 감사

>적용 대상: Windows Server 2019

방화벽 감사는 Windows Server 2019의 SDN 방화벽에 대 한 새로운 기능입니다. SDN 방화벽을 사용 하도록 설정 하면 모든 흐름 로깅을 사용 하도록 설정한는 SDN 방화벽 규칙 (Acl)에 의해 처리 기록 가져옵니다. 일치 하는 구문에서 로그 파일이 있어야 합니다 [Azure Network Watcher 흐름 로그](https://docs.microsoft.com/azure/network-watcher/network-watcher-nsg-flow-logging-overview)합니다. 이러한 로그는 진단에 사용 되는 또는 보관 나중에 분석할 수 있습니다. 

곧 Power BI와 같은 도구를 사용 하 여 이러한 파일을 처리 하는 방법의 예로 제공 됩니다.

_**사용해 보기 및 사용자 의견을 제공!**_

Hyper-v 호스트에 대 한 감사 하는 방화벽을 사용 하도록 설정 하는 샘플 스크립트는 다음과 같습니다. 시작 부분에서 변수를 업데이트 하 고 RSAT NetworkController 기능이 설치 된 Windows Server 2019 컴퓨터에서 실행이 키를 누릅니다.

```PowerShell
$logpath = "C:\test\log1"
$servers = @("sa18n22-2", "sa18n22-3", "sa18n22-4")
$uri = "https://sa18n22sdn.sa18.nttest.microsoft.com"

# Create log directories on the hosts
invoke-command -Computername $servers  {
    param(
        $Path
    )
    mkdir $path    -force
} -argumentlist $LogPath

# Set firewall auditing settings on Network Controller
$AuditProperties = new-object Microsoft.Windows.NetworkController.AuditingSettingsProperties
$AuditProperties.OutputDirectory = $logpath
set-networkcontrollerauditingsettingsconfiguration -connectionuri $uri -properties $AuditProperties -force  | out-null

# Enable logging on each server
$servers = get-networkcontrollerserver -connectionuri $uri
foreach ($s in $servers) {
    $s.properties.AuditingEnabled = @("Firewall")
    new-networkcontrollerserver -connectionuri $uri -resourceid $s.resourceid -properties $s.properties -force | out-null
}
```
