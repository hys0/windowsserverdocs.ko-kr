# <a name="windows-time-for-traceability"></a>추적 기능에 대 한 Windows 시간
>적용 대상: Windows Server 2016 버전 1709 이상 및 Windows 10 버전 1703 이상이


많은 섹터의 규정 시스템을 UTC로 추적 가능 해야 합니다.  즉, UTC와 관련 하 여 시스템의 오프셋을 증명할 수 있습니다.  규정 준수 시나리오를 사용 하려면 Windows 10 (버전 1703 이상) 및 Windows Server 2016 (버전 1709 이상) 수행 된 작업의 이해를 운영 체제의 관점에서 그림을 제공 하는 새 이벤트 로그 제공 됩니다. 시스템 시계입니다.  이러한 이벤트 로그 Windows 시간 서비스에 대해 지속적으로 생성 된 검사 또는 보관 나중에 분석할 수 있습니다.

이러한 새 이벤트는 다음 질문에 대답할 수을 사용 하도록 설정:

* 시스템 클록 변경 되었습니다
* 클록 주파수 수정한
* Windows 시간 서비스 구성을 수정한

## <a name="availability"></a>가용성

이러한 향상 된이 기능은 Windows 10 버전 1703 이상 및 Windows Server 2016 버전 1709 이상에 포함 됩니다.

## <a name="configuration"></a>Configuration

구성 없이이 기능을 실현 하는 데 필요 합니다.  이러한 이벤트 로그는 기본적으로 사용 하 고 있습니다 이벤트 뷰어에서 합니다 **응용 프로그램 및 서비스 Log\Microsoft\Windows\Time Service\Operational** 채널입니다.


*클럭 속도*
```

```
